# 岗位任务结构升级说明

## 📋 升级概述

本次升级将岗位学习任务从**粗粒度的一次性任务**优化为**细粒度的阶段性学习流程**，按照"理论-实践-输出"的科学学习路径设计，并将金币奖励从一口价改为**按月计算**。

## 🔄 学习流程设计

### 三阶段学习法

每个技能要求（requirement）都拆分为 3 个子任务：

#### 1️⃣ 理论学习（Theory）
- **目标**：建立知识体系，理解核心概念
- **内容**：
  - 学习基础理论和核心概念
  - 阅读官方文档和经典书籍
  - 观看视频教程和课程
- **产出**：学习笔记、思维导图
- **时长**：通常 1-4 周
- **金币占比**：30-40%

#### 2️⃣ 实践练习（Practice）
- **目标**：通过实战巩固知识，积累经验
- **内容**：
  - 完成练习题和小项目
  - 参与实战项目开发
  - 解决实际问题
- **产出**：代码作品、项目案例
- **时长**：通常 3-7 周
- **金币占比**：40-50%

#### 3️⃣ 成果输出（Output）
- **目标**：总结经验，形成可展示的成果
- **内容**：
  - 整理作品集和项目文档
  - 编写技术博客和教程
  - 发布开源项目
- **产出**：作品集、技术文章、开源项目
- **时长**：通常 1 周
- **金币占比**：10-20%

## 💰 金币奖励机制

### 旧机制（一口价）
```
完成任务 → 获得 18000 金币（一次性）
```
**问题**：
- ❌ 不符合真实工作场景（工资是按月发放）
- ❌ 无法体现学习过程的持续性
- ❌ 缺乏阶段性激励

### 新机制（按月计算）
```
学习第1个月 → 获得 3000 金币
学习第2个月 → 获得 3000 金币
...
学习第6个月 → 获得 3000 金币
总计：18000 金币/月 × 6 个月 = 108000 金币
```
**优势**：
- ✅ 符合真实工作场景（月薪制）
- ✅ 体现学习过程的持续性
- ✅ 提供阶段性激励和成就感
- ✅ 可以根据学习进度灵活调整

## 📊 示例对比

### 前端工程师 - HTML/CSS/JS 技能

#### 旧版本
```
技能：精通 HTML5/CSS3/ES6+
奖励：+3000元（一口价）
任务：完成 30 个页面布局练习 + 通过认证 + 手写方法
```

#### 新版本
```
技能：精通 HTML5/CSS3/ES6+
月奖励：3000 金币/月
学习周期：2 个月
总金币：6000 金币

子任务1：理论学习（3周，1000金币/月）
  - 学习 HTML5 语义化标签、表单、多媒体
  - 掌握 CSS3 选择器、盒模型、定位
  - 学习 Flexbox 和 Grid 布局
  - 掌握 ES6+ 核心语法
  - 学习 Promise、async/await

子任务2：实践练习（4周，1500金币/月）
  - 完成 30 个页面布局练习
  - 手写 10 个 JS 原生方法
  - 完成 5 个响应式网页
  - 通过 FreeCodeCamp 认证
  - LeetCode 简单题 50 道

子任务3：成果输出（1周，500金币/月）
  - 搭建个人作品集网站
  - 输出 5 篇技术博客
  - 整理学习笔记和代码库
  - GitHub 提交 30+ 天记录
```
